---
title: Update profile
description: Update the authenticated user's profile information
---

## Endpoint

```
PUT /api/users/profile
```

## Authentication

This endpoint requires authentication. Include a valid JWT token in the Authorization header:

```
Authorization: Bearer <token>
```

## Request body

<ParamField name="name" type="string">
  User's full name. Must be a non-empty trimmed string.
</ParamField>

<ParamField name="phoneNumber" type="string">
  User's phone number. Can be set to empty string or null to remove.
</ParamField>

<Note>
All fields are optional. Only include the fields you want to update. Other profile fields (email, role, register_number, employee_id, etc.) cannot be updated through this endpoint.
</Note>

## Response

### Success response

Returns the updated user profile data.

<ResponseField name="success" type="boolean" required>
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string" required>
  Success message
</ResponseField>

<ResponseField name="data" type="object" required>
  Updated user profile data
  
  <ResponseField name="id" type="string" required>
    User's unique identifier
  </ResponseField>
  
  <ResponseField name="name" type="string" required>
    User's full name
  </ResponseField>
  
  <ResponseField name="email" type="string" required>
    User's email address
  </ResponseField>
  
  <ResponseField name="role" type="string" required>
    User's role. One of: `student`, `faculty`, or `store_employee`
  </ResponseField>
  
  <ResponseField name="register_number" type="string">
    Student or faculty registration number (null if not applicable)
  </ResponseField>
  
  <ResponseField name="employee_id" type="string">
    Employee ID for store employees (null if not applicable)
  </ResponseField>
  
  <ResponseField name="phone_number" type="string">
    User's phone number (null if not provided)
  </ResponseField>
  
  <ResponseField name="is_email_verified" type="boolean" required>
    Whether the user's email has been verified
  </ResponseField>
  
  <ResponseField name="no_show_count" type="number" required>
    Number of times the user failed to pick up orders
  </ResponseField>
  
  <ResponseField name="trust_tier" type="string" required>
    User's trust level based on order history. One of: `good`, `watch`, or `restricted`
  </ResponseField>
  
  <ResponseField name="ordering_restricted_until" type="string">
    ISO 8601 date when ordering restrictions will be lifted (null if not restricted)
  </ResponseField>
  
  <ResponseField name="last_no_show_at" type="string">
    ISO 8601 date of the last no-show incident (null if none)
  </ResponseField>
  
  <ResponseField name="created_at" type="string" required>
    ISO 8601 date when the account was created
  </ResponseField>
  
  <ResponseField name="updated_at" type="string" required>
    ISO 8601 date when the profile was last updated
  </ResponseField>
  
  <ResponseField name="store" type="object">
    Store information (only present if user role is `store_employee` and owns a store)
  </ResponseField>
</ResponseField>

### Error responses

<ResponseField name="success" type="boolean" required>
  Always `false` for errors
</ResponseField>

<ResponseField name="message" type="string" required>
  Error description
</ResponseField>

## Validation rules

- `name`: Must be a non-empty string if provided. Whitespace is automatically trimmed.
- `phoneNumber`: Can be any string value, empty string, or null.
- Fields not in the allowed list (`name`, `phoneNumber`) are ignored.
- The `email`, `role`, `register_number`, `employee_id`, and other system fields cannot be modified through this endpoint.

## Examples

<CodeGroup>

```bash curl
curl -X PUT https://api.campusbite.com/api/users/profile \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jane Smith",
    "phoneNumber": "+1987654321"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.campusbite.com/api/users/profile', {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Jane Smith',
    phoneNumber: '+1987654321'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.put(
    'https://api.campusbite.com/api/users/profile',
    headers={
        'Authorization': 'Bearer YOUR_JWT_TOKEN',
        'Content-Type': 'application/json'
    },
    json={
        'name': 'Jane Smith',
        'phoneNumber': '+1987654321'
    }
)

data = response.json()
print(data)
```
