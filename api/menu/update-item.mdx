---
title: Update menu item
description: Update an existing menu item's details including name, price, description, category, availability, and image
---

## Endpoint

```
PUT /api/menu/:id
```

## Authentication

This endpoint requires authentication and store employee authorization.

<Warning>
  Only the owner of the store that owns this menu item can update it. Attempting to update another store's menu item will result in a 403 Forbidden error.
</Warning>

## Request

This endpoint accepts `multipart/form-data` for image uploads.

### Path parameters

<ParamField path="id" type="string" required>
  MongoDB ObjectId of the menu item to update
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `multipart/form-data`
</ParamField>

### Body parameters

All body parameters are optional. Only include the fields you want to update.

<ParamField body="name" type="string">
  Name of the menu item. Will be trimmed of whitespace.
</ParamField>

<ParamField body="price" type="number">
  Price of the menu item. Must be a positive number (minimum 0).
</ParamField>

<ParamField body="description" type="string">
  Description of the menu item. Pass empty string or null to clear.
</ParamField>

<ParamField body="category" type="string">
  Category for the menu item. Pass empty string or null to clear.
</ParamField>

<ParamField body="is_available" type="boolean">
  Availability status. Accepts boolean or string "true"/"false".
</ParamField>

<ParamField body="image" type="file">
  New image file for the menu item. This will replace the existing image.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable message describing the result
</ResponseField>

<ResponseField name="data" type="object">
  Contains the updated menu item data
  
  <ResponseField name="data.menuItem" type="object">
    The updated menu item object
    
    <ResponseField name="data.menuItem.id" type="string">
      Unique identifier for the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.store_id" type="string">
      ID of the store this item belongs to
    </ResponseField>
    
    <ResponseField name="data.menuItem.name" type="string">
      Name of the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.description" type="string">
      Description of the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.price" type="number">
      Price of the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.image_url" type="string">
      URL to the menu item image
    </ResponseField>
    
    <ResponseField name="data.menuItem.category" type="string">
      Category of the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.is_available" type="boolean">
      Availability status
    </ResponseField>
    
    <ResponseField name="data.menuItem.created_at" type="string">
      ISO 8601 timestamp of creation
    </ResponseField>
    
    <ResponseField name="data.menuItem.updated_at" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
  </ResponseField>
</ResponseField>

## Examples

<CodeGroup>

```bash cURL - Update price and availability
curl -X PUT https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -F "price=9.99" \
  -F "is_available=true"
```

```bash cURL - Update with new image
curl -X PUT https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -F "name=Deluxe Cheeseburger" \
  -F "description=Our upgraded burger with double cheese and bacon" \
  -F "price=11.99" \
  -F "image=@/path/to/new-burger.jpg"
```

```bash cURL - Update all fields
curl -X PUT https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -F "name=Premium Burger" \
  -F "description=The best burger in town" \
  -F "price=12.99" \
  -F "category=Premium Items" \
  -F "is_available=true"
```

```javascript JavaScript
const formData = new FormData();
formData.append('price', '9.99');
formData.append('is_available', 'true');

const response = await fetch('https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011', {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  },
  body: formData
});

const data = await response.json();
```

```python Python
import requests

url = "https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011"
headers = {
    "Authorization": "Bearer YOUR_ACCESS_TOKEN"
}
data = {
    "name": "Premium Burger",
    "description": "The best burger in town",
    "price": "12.99",
    "category": "Premium Items",
    "is_available": "true"
}

response = requests.put(url, headers=headers, data=data)
print(response.json())
```

### Error responses

#### 404 - Menu item not found

```json
{
  "success": false,
  "message": "Menu item not found."
}
```

#### 403 - Not authorized

<Warning>
  This error occurs when you try to update a menu item that belongs to another store.
</Warning>

```json
{
  "success": false,
  "message": "You are not authorized to update this menu item."
}
```

#### 401 - Unauthorized

```json
{
  "success": false,
  "message": "Authentication required."
}
```
