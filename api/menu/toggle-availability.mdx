---
title: Toggle menu item availability
description: Quick endpoint to toggle a menu item's availability status between available and unavailable
---

## Endpoint

```
PATCH /api/menu/:id/availability
```

## Authentication

This endpoint requires authentication and store employee authorization.

<Note>
  This is a convenience endpoint that toggles the `is_available` field. If the item is currently available, it becomes unavailable, and vice versa. You don't need to specify the new state.
</Note>

## Request

### Path parameters

<ParamField path="id" type="string" required>
  MongoDB ObjectId of the menu item to toggle
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

<Note>
  This endpoint does not require a request body. It automatically toggles the current availability state.
</Note>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable message describing the result. Will say either "Menu item is now available." or "Menu item is now unavailable."
</ResponseField>

<ResponseField name="data" type="object">
  Contains the updated menu item data
  
  <ResponseField name="data.menuItem" type="object">
    The updated menu item object
    
    <ResponseField name="data.menuItem.id" type="string">
      Unique identifier for the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.store_id" type="string">
      ID of the store this item belongs to
    </ResponseField>
    
    <ResponseField name="data.menuItem.name" type="string">
      Name of the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.description" type="string">
      Description of the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.price" type="number">
      Price of the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.image_url" type="string">
      URL to the menu item image
    </ResponseField>
    
    <ResponseField name="data.menuItem.category" type="string">
      Category of the menu item
    </ResponseField>
    
    <ResponseField name="data.menuItem.is_available" type="boolean">
      New availability status (toggled from previous state)
    </ResponseField>
    
    <ResponseField name="data.menuItem.created_at" type="string">
      ISO 8601 timestamp of creation
    </ResponseField>
    
    <ResponseField name="data.menuItem.updated_at" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
  </ResponseField>
</ResponseField>

## Examples

<CodeGroup>

```bash cURL
curl -X PATCH https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011/availability \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const response = await fetch('https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011/availability', {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  }
});

const data = await response.json();
console.log(data.message); // "Menu item is now available." or "Menu item is now unavailable."
```

```python Python
import requests

url = "https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011/availability"
headers = {
    "Authorization": "Bearer YOUR_ACCESS_TOKEN"
}

response = requests.patch(url, headers=headers)
result = response.json()
print(result["message"])
```

```java Java
HttpClient client = HttpClient.newHttpClient();

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.campusbite.com/api/menu/507f1f77bcf86cd799439011/availability"))
    .header("Authorization", "Bearer YOUR_ACCESS_TOKEN")
    .method("PATCH", HttpRequest.BodyPublishers.noBody())
    .build();

HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
System.out.println(response.body());
```

### Error responses

#### 404 - Menu item not found

```json
{
  "success": false,
  "message": "Menu item not found."
}
```

#### 403 - Not authorized

<Warning>
  This error occurs when you try to toggle availability for a menu item that belongs to another store.
</Warning>

```json
{
  "success": false,
  "message": "You are not authorized to update this menu item."
}
```

#### 401 - Unauthorized

```json
{
  "success": false,
  "message": "Authentication required."
}
```

## Use cases

This endpoint is useful for:

- Quickly marking items as out of stock during busy periods
- Temporarily disabling seasonal items
- Managing item availability based on inventory
- Hiding items during preparation or menu updates

<Note>
  If you need to set a specific availability state, use the [update menu item](/api/menu/update-item) endpoint with the `is_available` parameter instead.
</Note>
