---
title: Get menu
description: Retrieve the menu items for a specific store with optional filtering
---

## Endpoint

```
GET /stores/:id/menu
```

Retrieve all menu items for a specific store. Supports filtering by category and searching by name.

## Authentication

No authentication required. This is a public endpoint.

## Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the store (MongoDB ObjectId)
</ParamField>

## Query parameters

<ParamField query="category" type="string">
  Filter menu items by category (exact match)
</ParamField>

<ParamField query="search" type="string">
  Search menu items by name (case-insensitive partial match)
</ParamField>

## Response

<ResponseField name="success" type="boolean" required>
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string">
  Error message (only present when success is false)
</ResponseField>

<ResponseField name="data" type="object">
  Contains the response data (only present when success is true)
  
  <Expandable title="data properties">
    <ResponseField name="menuItems" type="array" required>
      Array of menu item objects, sorted by category and name
      
      <Expandable title="menu item object">
        <ResponseField name="id" type="string" required>
          Unique identifier for the menu item
        </ResponseField>
        
        <ResponseField name="_id" type="string" required>
          MongoDB ObjectId (same as id)
        </ResponseField>
        
        <ResponseField name="store_id" type="string" required>
          ID of the store this item belongs to
        </ResponseField>
        
        <ResponseField name="storeId" type="string" required>
          Store ID (camelCase)
        </ResponseField>
        
        <ResponseField name="name" type="string" required>
          Menu item name
        </ResponseField>
        
        <ResponseField name="description" type="string">
          Menu item description
        </ResponseField>
        
        <ResponseField name="price" type="number" required>
          Price of the item (minimum 0)
        </ResponseField>
        
        <ResponseField name="image_url" type="string">
          URL to menu item image
        </ResponseField>
        
        <ResponseField name="imageUrl" type="string">
          Menu item image URL (camelCase)
        </ResponseField>
        
        <ResponseField name="category" type="string">
          Category of the menu item (e.g., "Beverages", "Snacks")
        </ResponseField>
        
        <ResponseField name="is_available" type="boolean" required>
          Whether the item is currently available for ordering
        </ResponseField>
        
        <ResponseField name="isAvailable" type="boolean" required>
          Availability status (camelCase)
        </ResponseField>
        
        <ResponseField name="created_at" type="string">
          ISO 8601 timestamp of creation
        </ResponseField>
        
        <ResponseField name="createdAt" type="string">
          Creation timestamp (camelCase)
        </ResponseField>
        
        <ResponseField name="updated_at" type="string">
          ISO 8601 timestamp of last update
        </ResponseField>
        
        <ResponseField name="updatedAt" type="string">
          Last update timestamp (camelCase)
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Example requests

### Get all menu items

<CodeGroup>
```bash cURL
curl https://api.campusbite.com/stores/507f1f77bcf86cd799439011/menu
```

```javascript JavaScript
const storeId = '507f1f77bcf86cd799439011';
const response = await fetch(`https://api.campusbite.com/stores/${storeId}/menu`);
const data = await response.json();
```

```python Python
import requests

store_id = '507f1f77bcf86cd799439011'
response = requests.get(f'https://api.campusbite.com/stores/{store_id}/menu')
data = response.json()
```
</CodeGroup>

### Filter by category

<CodeGroup>
```bash cURL
curl "https://api.campusbite.com/stores/507f1f77bcf86cd799439011/menu?category=Beverages"
```

```javascript JavaScript
const storeId = '507f1f77bcf86cd799439011';
const category = 'Beverages';
const response = await fetch(
  `https://api.campusbite.com/stores/${storeId}/menu?category=${encodeURIComponent(category)}`
);
const data = await response.json();
```

```python Python
import requests

store_id = '507f1f77bcf86cd799439011'
params = {'category': 'Beverages'}
response = requests.get(
    f'https://api.campusbite.com/stores/{store_id}/menu',
    params=params
)
data = response.json()
```
</CodeGroup>

### Combine filters

<CodeGroup>
```bash cURL
curl "https://api.campusbite.com/stores/507f1f77bcf86cd799439011/menu?category=Beverages&search=iced"
```

```javascript JavaScript
const storeId = '507f1f77bcf86cd799439011';
const params = new URLSearchParams({
  category: 'Beverages',
  search: 'iced'
});
const response = await fetch(
  `https://api.campusbite.com/stores/${storeId}/menu?${params}`
);
const data = await response.json();
```

```python Python
import requests

store_id = '507f1f77bcf86cd799439011'
params = {
    'category': 'Beverages',
    'search': 'iced'
}
response = requests.get(
    f'https://api.campusbite.com/stores/{store_id}/menu',
    params=params
)
data = response.json()
```
</CodeGroup>

## Notes

<Note>
**Sorting:** Menu items are automatically sorted first by category (alphabetically), then by name (alphabetically) within each category.
</Note>

<Note>
**Search behavior:** The search parameter performs a case-insensitive partial match on the menu item name. For example, searching for "coffee" will match "Iced Coffee", "Coffee Latte", etc.
</Note>

<Note>
**Availability:** The endpoint returns all menu items regardless of their `is_available` status. Client applications should check the `is_available` or `isAvailable` field to determine if an item can be ordered.
</Note>

<Note>
**Empty results:** If no menu items match the filters or if the store has no menu items, the endpoint returns an empty array rather than an error.
</Note>
