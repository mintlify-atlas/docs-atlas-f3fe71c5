---
title: Get order details
description: Retrieve detailed information about a specific order
---

## Overview

Retrieves complete details for a single order. Access is role-based:

- **Customers**: Can only view their own orders
- **Store employees**: Can only view orders for their store

## Authentication

Requires authentication with JWT token. Available to all authenticated users.

## Endpoint

```
GET /api/orders/:id
```

## Path parameters

<ParamField path="id" type="string" required>
  The unique order ID (MongoDB ObjectId)
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  Complete order details
  
  <Expandable title="Order properties">
    <ResponseField name="id" type="string">
      Unique order identifier
    </ResponseField>
    
    <ResponseField name="orderNumber" type="string">
      Human-readable order number
    </ResponseField>
    
    <ResponseField name="paymentReference" type="string">
      Unique payment reference starting with `CBPAY`
    </ResponseField>
    
    <ResponseField name="orderStatus" type="string">
      Current order status: `placed`, `accepted`, `processing`, `ready`, `picked_up`, or `cancelled`
    </ResponseField>
    
    <ResponseField name="paymentStatus" type="string">
      Payment status: `pending`, `success`, or `failed`
    </ResponseField>
    
    <ResponseField name="paymentMethod" type="string">
      Payment method used (e.g., `direct_store_upi`)
    </ResponseField>
    
    <ResponseField name="transactionId" type="string">
      UPI transaction ID if provided
    </ResponseField>
    
    <ResponseField name="totalAmount" type="number">
      Total order amount in rupees
    </ResponseField>
    
    <ResponseField name="items" type="array">
      Array of ordered items with menuItemId, name, price, quantity, and total
    </ResponseField>
    
    <ResponseField name="specialInstructions" type="string">
      Customer's special instructions if provided
    </ResponseField>
    
    <ResponseField name="isCommitmentConfirmed" type="boolean">
      Whether customer confirmed they're on the way
    </ResponseField>
    
    <ResponseField name="commitmentConfirmedAt" type="string">
      ISO timestamp when commitment was confirmed
    </ResponseField>
    
    <ResponseField name="commitmentDeadlineAt" type="string">
      ISO timestamp when commitment confirmation expires
    </ResponseField>
    
    <ResponseField name="readyAt" type="string">
      ISO timestamp when order was marked as ready
    </ResponseField>
    
    <ResponseField name="readyExpiresAt" type="string">
      ISO timestamp when ready order expires (20 minutes from ready)
    </ResponseField>
    
    <ResponseField name="cancelledAt" type="string">
      ISO timestamp when order was cancelled
    </ResponseField>
    
    <ResponseField name="cancellationReason" type="string">
      Reason for cancellation: `payment_timeout`, `payment_failed`, or `no_show_timeout`
    </ResponseField>
    
    <ResponseField name="isOtpVerified" type="boolean">
      Whether the pickup OTP was verified
    </ResponseField>
    
    <ResponseField name="store" type="object">
      Complete store details including id, name, location, contact info
    </ResponseField>
    
    <ResponseField name="customer" type="object">
      Customer details (only for store employee requests) including name, email, phone, register number
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO timestamp when order was created
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO timestamp when order was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

## Example request

<CodeGroup>
```bash cURL
curl -X GET https://api.campusbite.com/api/orders/65f7a8b9c1234567890abcde \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const orderId = '65f7a8b9c1234567890abcde';
const response = await fetch(
  `https://api.campusbite.com/api/orders/${orderId}`,
  {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
);

const data = await response.json();
```
</CodeGroup>

## Order status meanings

- **placed**: Order created, awaiting payment verification
- **accepted**: Payment confirmed, awaiting preparation
- **processing**: Order is being prepared by the store
- **ready**: Order is ready for pickup, OTP generated
- **picked_up**: Order collected by customer
- **cancelled**: Order cancelled due to payment failure, timeout, or no-show

<Note>
When an order reaches `ready` status, a 6-digit OTP is generated and sent to the customer. The store must verify this OTP when the customer arrives to collect the order.
</Note>
